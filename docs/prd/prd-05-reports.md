# PRD-05: Reports

## Goal

Allow Supervisors to generate accounting exports for any date range. Generation locks the included entries (preventing Member edits) and stores an immutable JSONB snapshot. Output is a CSV file streamed to the browser.

---

## Scope

- Date range picker
- Entry preview before generating
- Generate report: lock entries + store snapshot + stream CSV
- CSV format: summary block + detail rows, filterable by user and category
- Past reports list

---

## User Stories

**US-01** As a Supervisor, I want to select a date range and preview how many entries will be included so that I can confirm the scope before generating.

**US-02** As a Supervisor, I want to generate a report so that all included entries are locked and I receive a CSV file.

**US-03** As a Supervisor, I want to download a CSV broken down by user and category so that I can import it into payroll or accounting tools.

**US-04** As a Supervisor, I want to view a list of previously generated reports so that I can re-download earlier exports.

---

## Acceptance Criteria

### Preview (US-01)

- [ ] Report form has: period start date, period end date, optional user filter, optional category filter
- [ ] On date range selection, the UI shows a preview count ("X entries across Y members")
- [ ] Preview updates when filters change
- [ ] Start date must be ≤ end date — validation error otherwise

### Generate (US-02)

- [ ] Clicking "Generate" triggers a server action (not a client-side mutation)
- [ ] Server action runs in a transaction:
  - [ ] SELECTs unlocked entries in range (with applied filters)
  - [ ] INSERTs a `reports` row with `time_entries_snapshot` as JSONB
  - [ ] UPDATEs matched entries `SET is_locked = TRUE`
- [ ] CSV is streamed to the browser and triggers a file download
- [ ] On success, the new report appears in the past reports list
- [ ] If zero entries match, generation is blocked with an informative message

### CSV format (US-03)

- [ ] File name format: `report_YYYY-MM-DD_YYYY-MM-DD.csv`
- [ ] Summary block at top: one row per user+category combination with total minutes
- [ ] Blank separator row between summary and detail sections
- [ ] Detail section: one row per entry (date, user full name, category name, duration_minutes, note)
- [ ] Both sections include a header row
- [ ] All data sourced from the `time_entries_snapshot` (not live table re-query)

### Past reports (US-04)

- [ ] Reports list shows: period range, generated date, generated by (Supervisor name)
- [ ] Each row has a "Download CSV" action that regenerates the CSV from the stored snapshot
- [ ] Reports are sorted newest-first

---

## Schema / API Notes

- Tables: `reports`, `time_entries` — see [schema.md](../architecture/schema.md)
- Snapshot type: `TimeEntrySnapshot[]` — see [schema.md](../architecture/schema.md)
- Permissions: `reports:read`, `reports:write` (Supervisor only) — see [roles-permissions.md](../architecture/roles-permissions.md)
- Re-download: derive CSV from `reports.time_entries_snapshot` JSONB, joined against current `profiles` and `categories` for display names

---

## Out of Scope

- PDF or Excel output (CSV only in v1)
- Scheduled / automatic report generation
- Member access to reports
- Unlocking entries after a report is generated
- Per-`employment_type` report sections (data captured; UI hidden in v1)
